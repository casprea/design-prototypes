Requirement

Remote access is access to organizational systems (or processes acting on behalf of users) that communicate through the use of external networks, such as the Internet. Types of remote access include dial-up, broadband, and wireless. An organization can encrypt all transmissions of remote sessions using IETF compliant cryptographic algorithms as part of a network access control solution.

Implementation design
Generated by Vanta AI

To control access to our systems remotely, we set up a few layers of protection. Anyone working remotely must connect through our VPN, and everyone needs to verify their identity with a second factor before logging in. We also set each session can only stay active for some time out. For engineers who need more access our policy says anyone who needs access to a production system has been verified, and every access session is logged automatically. All communication is happening over encrypted channels. We also have simulated automated systems testing so nothing is flying under the radar.

Encryption at rest: backup environments

All customer data must be encrypted at rest using AES-256 or equivalent. This applies to primary storage, backup systems, databases, and data systems — including both primary storage and backup environments — and must be verifiable through automated testing.
